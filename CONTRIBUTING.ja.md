# OpenGIKAIへのコントリビューション

[🇬🇧 English](./CONTRIBUTING.md)

OpenGIKAIへの貢献に興味を持っていただきありがとうございます！このプロジェクトは、政治的中立性と透明性を維持しながら、国会の審議内容を誰にでもアクセスしやすくすることを目指しています。

## 行動規範

- 敬意を持ち、建設的であること
- 政治的中立性を維持すること — OpenGIKAIは透明性のためのツールであり、主張のためのプラットフォームではありません
- 正確さとアクセシビリティに注力すること

## コントリビューションの方法

### Issue の報告

- バグ報告や機能提案は[GitHub Issues](https://github.com/wharfe/open-gikai/issues)をご利用ください
- バグの場合は再現手順を含めてください
- 機能提案の場合はユースケースを説明してください

### プルリクエスト

1. リポジトリをフォーク
2. フィーチャーブランチを作成（`git checkout -b feature/your-feature`）
3. 変更を実装
4. Lintとテストを実行（`npm run lint`）
5. わかりやすいコミットメッセージでコミット
6. プッシュしてプルリクエストを作成

### 開発環境のセットアップ

```bash
git clone https://github.com/wharfe/open-gikai.git
cd open-gikai
npm install
npm run dev
```

## ガイドライン

### コード

- TypeScript strictモード
- コード内のコメントは英語
- ユーザー向けテキストは日本語
- スタイリングにはTailwind CSSを使用
- コードベースの既存パターンに従う

### AIプロンプト

AIプロンプト（要約、分類など）の変更には慎重なレビューが必要です：

- 変更の理由を説明すること
- 生成結果の変更前後の例を示すこと
- 政治的中立性が維持されていることを確認すること
- プロンプトの変更は少なくとも1人の他のコントリビューターによるレビューを必須とする

### コミット

- 明確でわかりやすいコミットメッセージを使用
- 該当するIssue番号を参照

## 協力が必要な分野

- フロントエンドコンポーネントとUI改善
- アクセシビリティ（a11y）の強化
- パフォーマンス最適化
- ドキュメントと翻訳
- AIプロンプトの改善（中立性の保証付き）
- テスト

## 質問がある場合

[GitHub Discussions](https://github.com/wharfe/open-gikai/discussions)またはIssueを作成してください。
